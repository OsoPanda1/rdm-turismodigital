Nodo Cero = kernel operativo del territorio Real del Monte dentro de la red federada TAMV Online.

Capas:
1. Orquestación: API gateway + workers + schedulers (Edge Functions Supabase como sustituto de K8s en Lovable).
2. Datos: Postgres (Supabase) como fuente única de verdad por dominio (identity, tourism, commerce, civic, analytics, observability).
3. Integración: bus de eventos (postgres_changes/Realtime) + webhooks (stripe-webhook, github-sync) + conectores entrantes (telemetry-ingest, places-register).
4. Servicios de dominio: Identidad Ciudadana (profiles+user_roles), Catálogo turístico (events, businesses), Comercios (businesses con monetización), Campañas (analytics), Gobierno de datos (RLS + audit_signatures).
5. Frontends: Nodo Cero Control (/dashboard), Portal público (/), Panel comercio (/comercio/registro), Repositorio federado (/repositorio).

Funciones clave:
- Resolución de identidad unificada (ciudadano/turista/comercio/sistema)
- Core de datos urbanos (vista única por dominio)
- Motor de orquestación de experiencias (descubrimiento→interés→transacción→fidelización)
- Integración con IA (Isabella vía isabella-chat, kernel-isabella-chat con SSE)
- Seguridad por diseño: RLS, JWT en edge functions, firma Stripe, audit_signatures

Relación con TAMV: RDM = nodo territorial 1; TAMV = backbone federado heptafederal (7 federaciones). Nodo Cero expone /federation-state al backbone.

Casos de uso eventos tradicionales:
- Procesiones, rutas, horarios → tabla events + map layers
- Comercios temporales del evento → businesses con activación Stripe temporal
- Insignias por participación → user_roles extendido
- Analítica de afluencia → analytics + tamvcrums_logs (ECG urbano)
